# 10 — アイテム一覧画面

## 概要

登録済みアイテムの棚卸しビュー。「何を飲んでいるか」を整理・確認する画面。2番目のタブ。

## 参照

- REQUIREMENTS.md §5.3（アイテム一覧画面）

## タスク

- [x] `app/(tabs)/items.tsx` に一覧を実装
  - カテゴリ別セクション表示（お薬／サプリ／その他）
  - 各行：名前／カテゴリ色ドット／服用パターンの日本語表示（例「3日に1回」「月・木」「毎日」「不定期」）／最終服用日（`getLastTakenDate`）
- [x] 行タップ → 編集画面（06の `app/item/[id]`）へ遷移
- [x] 服用中止（is_active=0）の操作
  - 中止しても履歴は残る旨が伝わるUI文言にする
  - 中止済みアイテムは薄い表示（textMuted）で下部にまとめる or 折りたたむ（再開＝is_active=1 も可能に）
- [x] 新規登録への導線（ヘッダー右の「＋」ボタン → `app/item/new`）
- [x] 空状態：アイテム0件時は登録を促す表示

## デザイン制約

- リスト項目は本文 14/400、補足 10–11/400 textSecondary
- カード角丸14・影なし・区切りは border 色の細線

## 完了条件

- 登録済みアイテムがカテゴリ別に表示され、編集・中止・再開ができる
- 中止したアイテムの過去の記録がカレンダーに残っている（履歴保持の確認）

## 依存

- 04（データアクセス層）、06（アイテム登録・編集画面）
