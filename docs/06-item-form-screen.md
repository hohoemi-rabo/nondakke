# 06 — アイテム登録・編集画面

## 概要

薬・サプリの登録/編集フォーム。**30秒で登録完了**が目標（競合の低評価要因＝入力項目過多への対策）。必須は「名前」と「服用パターン」の2つだけ。

## 参照

- REQUIREMENTS.md §5.4（アイテム登録・編集画面）
- DESIGN.md §5「登録フォーム」

## タスク

- [ ] ルート作成：`app/item/new.tsx`（新規）と `app/item/[id].tsx`（編集）。モーダル表示で開く
- [ ] フォーム構成（上から順に）
  - **名前**（必須）：プレースホルダー「例：血圧の薬、ビタミンD など自分がわかる名前でOK」
  - **服用パターン**（必須）：4択チップ（毎日／N日に1回／曜日指定／不定期）
    - 「N日に1回」選択時のみ日数入力を表示（例：3日に1回）
    - 「曜日指定」選択時のみ曜日選択（月〜日の複数選択）を表示
  - カテゴリ（任意・デフォルト「お薬」）：3択チップ
  - タイミング（任意）：朝／昼／夕／就寝前の複数選択チップ
  - メモ（任意）：自由記入（用量・注意事項など）
- [ ] バリデーション：名前が空・interval で日数未入力・weekly で曜日未選択のとき保存不可（エラーはインラインで表示）
- [ ] 保存で `createItem` / `updateItem` を呼び、前の画面に戻る
- [ ] 編集時は既存値をフォームに反映

## デザイン制約

- 必須2項目を最上部、任意項目は下に寄せる
- 選択中チップ = accentLight 背景＋accentDark 文字
- 薬名データベース・検索機能は作らない（スコープ外）

## 完了条件

- 名前入力→パターン選択→保存 の最短フローが3操作程度で完了する
- 新規登録・編集・バリデーションエラーの各ケースが動作する
- `npm run lint` / `npx tsc --noEmit` / `npm test` が通る
