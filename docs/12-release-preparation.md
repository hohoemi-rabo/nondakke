# 12 — リリース準備（EAS Build / Google Play）

## 概要

Google Play での一般公開（完全無料）に向けたビルド・ストア申請準備。Android 先行、iOS はストア申請せずコードのクロスプラットフォーム性のみ維持。

## 参照

- REQUIREMENTS.md §1（配布）、§2（ビルド/配布）、§7（非機能要件）

## タスク

### アプリアセット

- [ ] アプリアイコン・アダプティブアイコンを「のんだっけ？」用に差し替え（現在はテンプレートのまま。`app.json` の adaptiveIcon 背景色 `#E6F4FE` もブランド色に合わせ見直し）
- [ ] スプラッシュ画面の画像・背景色を調整（白ベース）
- [ ] ストア用スクリーンショット・説明文の準備
  - 説明文に明記：ネットワーク通信なし（プライバシー訴求）／医療アドバイスは行わない旨

### ビルド設定

- [ ] `app.json` に `android.package`（アプリケーションID）と `versionCode` を設定
- [ ] EAS Build のセットアップ（`eas.json` 作成、production プロファイル）
- [ ] 権限の確認：**不要な権限が AndroidManifest に含まれないこと**（ネットワーク権限も原則不要の方針。ビルド後に実際のマニフェストを確認）

### Google Play 申請要件

- [ ] プライバシーポリシーの公開URL（Play Console 要件。GitHub Pages 等の静的ホスティングで可）
- [ ] データセーフティフォームの申告：データ収集なし・第三者共有なし
- [ ] コンテンツレーティング・対象年齢の設定

### 最終確認

- [ ] 実機での動作確認（記録の主要フロー一通り＋アプリ再起動でのデータ永続確認）
- [ ] `npm run lint` / `npx tsc --noEmit` / `npm test` すべてグリーン

## 完了条件

- EAS Build で production AAB がビルドでき、実機にインストールして全機能が動作する
- Play Console の内部テストトラックにアップロードできる状態
