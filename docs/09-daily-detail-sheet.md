# 09 — 日別詳細シート・記録機能

## 概要

日セルタップで下から出るシート。その日の予定・記録を一覧し、ワンタップで「飲んだ」を記録する。**過去日の追記・修正・削除**が主要ユースケース（アプリ名「のんだっけ？」の核心体験）。

## 参照

- REQUIREMENTS.md §5.2（日別詳細シート）、§6-3（記録のワンタップ登録）
- DESIGN.md §5「記録ボタン」

## タスク

- [ ] ボトムシートを実装（選択日を受け取って表示）
  - 影は elevation 2 相当まで、角丸 sheet(20)
- [ ] その日の予定アイテムを表示（05の導出ロジックを使用）
  - タイミング（朝・昼・夕・就寝前）ごとに記録ボタンを表示。タイミング未設定のアイテムは1日1回として `timing='none'` で記録
- [ ] 記録ボタン（角丸カード型の大ボタン）
  - 未記録＝白背景＋border枠＋「のんだ！」、記録済み＝accentLight背景＋accentチェック
  - タップで即時に見た目が切り替わる。**確認ダイアログは出さない**。取り消しは再タップ（削除）
  - タップ時に expo-haptics で軽いフィードバック（導入済みパッケージ）
- [ ] 過去日の操作
  - 過去日でも同じUIで追記・取り消しができる（当日と操作を変えない）
  - 未来日は記録不可（ボタンを非活性 or 非表示。実装judgment だが飲んでいないものを記録させない）
- [ ] as_needed（不定期）アイテムの記録
  - 予定に出ないため、シート内に「不定期のアイテムを記録」導線を置き、任意日に記録追加できる
- [ ] 記録の変更をカレンダー・サマリーカードに即時反映

## 設計メモ

- interval 型は記録の追加・削除で以降の予定が自動でずれる（導出方式）。シートを閉じたときにカレンダーが再計算後の状態になっていること
- おめでとう系モーダル・過剰アニメーションは入れない（記録は3秒で終わる体験）

## 完了条件

- 当日・過去日の記録追加/取り消しがワンタップででき、UNIQUE制約による二重記録が起きない
- 過去日の記録操作後、interval 型の予定がカレンダー上で正しく再計算される
- as_needed アイテムを任意日に記録できる

## 依存

- 05（予定日導出ロジック）、07（カレンダー画面）
