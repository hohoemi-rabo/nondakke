# 07 — カレンダー画面（ホーム）

## 概要

起動時に直接表示されるホーム画面。月表示カレンダー＋サマリーカードで「飲む日」と「今日の状態」がパッと見でわかる。

## 参照

- REQUIREMENTS.md §5.1（ホーム＝カレンダー画面）
- DESIGN.md §5「サマリーカード」「カレンダー」

## タスク

- [x] `app/(tabs)/index.tsx` に月表示カレンダーを実装
  - カレンダーは自作する（外部カレンダーライブラリは導入しない — DESIGN.md の日セル仕様を満たすため。7×5〜6グリッドの自前実装で十分）
  - 月送り：ヘッダーの前月/翌月ボタン
  - 曜日ヘッダー：日=sunday色、土=saturday色、平日=textSecondary
- [x] 日セルの状態表示（05の `getMonthSchedule` を使用）
  - 予定あり：カテゴリ色ドット（5px、最大3個、4個以上は「+」）
  - 記録済み過去日：チェックマーク（accent）、日付数字は textMuted
  - 予定ありなのに未記録の過去日：ドットを輪郭のみ（塗りなし）
  - 今日：セル背景 accentLight・角丸 cell、日付数字ウェイト500
- [x] サマリーカード（カレンダー上部）
  - 「今日のむもの」ラベル＋大数字（件数）、右に「✓ のんだ n」「のこり n」
  - 全部飲み終えたら「今日はぜんぶのんだ！」に文言変更（数字ゼロを見せない）
  - 背景 accentLight、角丸 card、影なし
- [x] 凡例（お薬・サプリ・その他）をカレンダー直下に常時表示
- [x] 日セルタップ → 日別詳細シートを開く（シート本体は 09。ここではタップイベントと選択日の受け渡しまで）
- [x] データ取得：`listRecordsByMonth` で月単位一括取得（日ごとにクエリしない）。画面フォーカス時・記録変更時に再取得

## デザイン制約

- 日セルの見た目が小さくてもタップ判定は44×44以上
- アイテム未登録時は空状態表示（登録画面への導線を置く）

## 完了条件

- 予定・記録・未記録の3状態がカレンダー上で見分けられる
- 月送りで過去・未来の月も正しく予定が導出される
- サマリーカードの数字が記録操作に追随して更新される

## 依存

- 02（デザイントークン）、05（予定日導出ロジック）
