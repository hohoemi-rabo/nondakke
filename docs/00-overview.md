## 実装順序と依存関係

```
01 プロジェクト基盤整備
 └─ 02 デザイントークン・共通UI部品
 └─ 03 SQLiteデータベース基盤
     └─ 04 データアクセス層（リポジトリ）
         └─ 05 予定日導出ロジック（コア）
             ├─ 06 アイテム登録・編集画面
             ├─ 07 カレンダー画面（ホーム）
             │   └─ 08 カテゴリ切替スワイプ
             │   └─ 09 日別詳細シート・記録機能
             ├─ 10 アイテム一覧画面
             └─ 11 設定画面
                 └─ 12 リリース準備（EAS Build / Google Play）
```

## 運用ルール

- 1チケット=1つの独立したPR/コミット単位を目安とする（大きい場合はチケット内タスク単位でコミット）
- チケット完了時は「完了条件」をすべて満たすこと
- 着手中に仕様の曖昧さを見つけたら、REQUIREMENTS.md / DESIGN.md を更新してからコードを書く
